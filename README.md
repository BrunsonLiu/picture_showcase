# 意识群岛 | Mind Archipelago

> 一个可访问的静态网页，用于展示 AI 协创图像，外化思维结构。

[在线预览](https://brunsonliu.github.io/picture_showcase/)

---

## 当前状态

这是一个单 HTML 文件项目，内嵌 CSS 和 JS，无构建工具。目前包含两个核心系统：

1. **浮岛系统**：可拖拽缩放的无限画布上悬浮着 7 个浮岛，点击、Enter 或空格可聚焦展开
2. **迷雾系统**：画布被雾覆盖，鼠标是探照灯，雾会缓慢回涌，打开浮岛可永久点亮
3. **体验兜底**：聚焦卡片提供显式关闭按钮，支持 Esc 退出、R/0 回到中心，并为远程图片加载失败提供渐变占位

## 技术路线

### 1. 无限画布

用一个 `#world` 容器，通过 `transform: translate(X, Y) scale(Z)` 实现视口平移和缩放。所有浮岛是 `#world` 内的 DOM 元素，绝对定位在画布坐标系中。

- **拖拽平移**：mousedown 记录起点，mousemove 计算偏移量应用到 translate
- **滚轮缩放**：以鼠标指针为中心缩放，通过数学补偿 `newPan = mouse - (mouse - oldPan) × (newZoom / oldZoom)` 实现
- **惯性缓动**：松手后根据最后几帧的速度继续滑动，每帧衰减 7%，低于阈值停止

### 2. 浮岛

每个浮岛是一个 DOM 元素，结构为 `.island > .island-bob > .island-glass > .island-thumb`。

- `.island-bob`：CSS animation 做上下微浮，每个岛延迟不同
- `.island-glass`：磨砂玻璃质感，`backdrop-filter: blur(12px)` + 半透明背景，不支持时降级为不透明深灰
- 不规则圆角 `border-radius: 18px 24px 20px 22px`
- 尺寸由数据中的 `scale` 字段决定：`宽 = 200 + scale × 40`
- **浮岛永远保持原始外观**，不使用任何 `filter: brightness()` 或 `opacity` 压暗。完全依赖上层雾 Canvas 的物理遮挡来制造"被雾挡住"的效果

### 3. 聚焦动画

点击浮岛时，创建一个独立的 `.focus-card` 元素（不在 `#world` 内），从浮岛的屏幕位置飞向屏幕中央。

**为什么 focus card 不放在 #world 里**：聚焦时 `#world` 会被 `filter: blur(8px)` 虚化，如果 card 在里面也会被模糊。放在独立的 `#focus-layer` 中可以保持清晰。

**动画流程**：
1. 原浮岛隐藏
2. 创建 focus card，定位在浮岛当前屏幕坐标
3. `void offsetHeight` 强制布局
4. 设置 CSS transition，改变 left/top/width/height 到目标位置
5. 双层 `requestAnimationFrame` 确保初始帧先渲染再触发动画
6. 420ms 后添加 `.expanded` 类展开内部内容

退出时逆向操作，用更快的缓动曲线（0.38s vs 0.45s）。

### 4. 迷雾系统

#### 4.1 雾层

一个全屏 Canvas（`#fog-canvas`），z-index 在浮岛之上、聚焦层之下。

**初始状态**：整个 Canvas 填充深灰蓝色 `rgb(14,18,28)`，完全遮挡下方内容。

**雾的纹理**：启动时程序生成一张 1024×1024 的"大团云雾"贴图。方法是在 Canvas 上绘制 15 个大尺寸径向渐变团块，每个团块的参数：

| 参数 | 范围 |
|------|------|
| 半径 | 160-320px |
| 透明度 | 0.35-0.7 |
| 色值 | `rgb(14,18,28)`（雾色） |

每个团块有 4 个 color stop（0→0.3→0.6→1），从实心到完全透明。15 个团块重叠后形成自然的云雾形态。

**为什么不用 Perlin 噪声**：之前用 3 层 FBM Perlin 噪声生成纹理，效果偏碎、偏"数字噪点"，不像大团云雾。改为大尺寸软渐变团块叠加后，纹理更像 PvZ 那种大团横移的云。

**雾的滚动**：3 层贴图以不同速度和方向做视差平移，用 `ctx.translate()` 偏移后平铺绘制。每层的参数：

| 层 | 水平速度 | 垂直速度 | 缩放 | 不透明度 |
|----|---------|---------|------|---------|
| 1  | 0.25    | 0.02    | 1.0  | 0.04   |
| 2  | -0.15   | 0.04    | 1.4  | 0.03   |
| 3  | 0.08    | -0.02   | 1.8  | 0.02   |

**当前局限**：贴图是程序生成的，不是美术手绘的。理想情况下应由 UI 设计师输出一张 1024×1024 或更大的无缝衔接云雾贴图（白底，带 Alpha 通道），替换当前的程序生成版本。程序生成的团块虽然比 Perlin 噪声更像云，但仍然缺少手绘贴图的那种有机细节。

#### 4.2 探照灯

鼠标位置周围，用 `destination-out` 合成模式绘制预生成的不规则遮罩图，清除雾。

**遮罩图生成**：启动时程序生成一张 512×512 的遮罩 Canvas。中心是一个主径向渐变（5 个 color stop，从纯白到透明），周围叠加 8 个偏移的小径向渐变（"絮状"边缘），模拟不规则的光晕边缘。

**旋转动画**：每帧 `spotlightAngle += 0.003`，绘制时先 `translate` 到鼠标位置，再 `rotate(spotlightAngle)`，然后 `drawImage(spotlightMaskTex)`。旋转让光晕边缘持续缓慢流动，产生"呼吸"感。

- 基础半径：桌面 180px，移动端 200px
- ±8px 的呼吸脉动
- 绘制时 globalAlpha 0.35，避免一次性清空太多雾

**为什么不用 createRadialGradient**：之前用 6 个偏移径向渐变叠加 + sin/cos wobble，但边缘仍然偏圆形。改用预生成遮罩图后，8 个絮状团块让边缘更不规则，加上旋转动画，光晕有了流动感。

**当前局限**：遮罩图同样是程序生成的。理想情况下应由设计师输出一张 256×256 的"不规则云朵边缘"遮罩图（中间实心，边缘呈絮状、散落状），替换当前版本。

#### 4.3 雾的回涌

每帧在全屏覆盖一层极低透明度的雾色 `rgba(14,18,28,0.003)`，同时探照灯区域用 `destination-out` 清除。当鼠标移走后，只有回涌在作用，雾逐渐恢复。

回涌速率从之前的 0.015 降到 0.003（5 倍差距），因为：
- 新的贴图式雾比旧 Perlin 噪声更不透明、更有体积感
- 降低回涌让用户有更多时间"看清"雾下的内容，减少压迫感
- 探照灯的清除速率（globalAlpha 0.35）远超回涌速率，所以探照灯区域始终清晰

#### 4.4 浮岛在雾中的状态

浮岛自身不做任何亮度/透明度调整，永远保持原始外观。视觉上的"被雾挡住"完全由上层雾 Canvas 的物理遮挡实现。

| 状态 | 视觉效果 | 实现方式 |
|------|---------|---------|
| 未点亮 | 被雾完全遮挡，几乎不可见 | 雾 Canvas 覆盖在浮岛上方 |
| 被探照灯照到 | 雾被部分清除，浮岛隐约可见 | `destination-out` 在鼠标位置清除雾 |
| 已点亮 | 雾被永久清除，浮岛完全可见 | 每帧以更强 `destination-out`（0.3）清除该位置 |

**颜色渗出**：在雾 Canvas 上，未点亮浮岛的位置绘制极淡的主色调径向渐变。这是雾的一部分，不是浮岛上的装饰。效果是雾在那个位置微微泛着浮岛的主色，给用户"这里有什么"的暗示。

#### 4.5 永久点亮与脉冲光环

点击浮岛聚焦时，数据中的 `revealed` 设为 true。之后每帧在雾 Canvas 上以更强的 `destination-out`（0.3 vs 回涌的 0.003）清除该位置，远超回涌速率，雾永远无法恢复。

**脉冲光环**：点亮瞬间，向 `revealPulses[]` 数组推入一个脉冲对象：

```javascript
{
  cx: 浮岛屏幕中心X,
  cy: 浮岛屏幕中心Y,
  maxR: 浮岛最大边 × 2.5,
  startTime: performance.now(),
  duration: 2000  // 2秒
}
```

每帧遍历脉冲数组，根据进度计算当前半径和透明度，用 `destination-out` 绘制旋转的遮罩图。效果是点亮瞬间有一个不规则的光环从浮岛中心向外扩散，把周围的雾"推开"——比简单挖洞更有仪式感。

全部 7 个浮岛点亮后，1.5 秒后雾永久消散（CSS opacity 过渡到 0），2 秒后隐藏浮岛"归一"从中央浮现。

### 5. 彩蛋

画布右下角 (780, 480) 有一个极淡的粉色光渗。探照灯照到该位置时触发计数，3 秒冷却。照到 3 次后，该位置变成隐藏浮岛"·"。

### 6. 关联线

聚焦时，如果作品数据中有 `relatedTo` 字段，在独立 Canvas 上绘制二次贝塞尔曲线。曲线用 `setLineDash` + 动态 `lineDashOffset` 做流动虚线，3 个光点沿曲线移动。

### 7. 移动端适配

- **划动（Swipe/Drag）**：单指在屏幕上滑动时，同时做两件事——平移画布 + 手指位置作为放大版探照灯（半径 200px）擦除迷雾。回涌速率进一步放慢，允许用户"抹出"一片视野
- **点击（Tap）**：位移小于 10px 且时间短于 300ms 判定为点击，直接触发对应位置浮岛的聚焦动画
- 双指捏合 → 画布缩放
- 聚焦时点击外部 → 退出
- 聚焦尺寸 85vw，文字上下堆叠
- 雾的 blur 降级为 4px（桌面 8px）

## 已知问题（诚实记录）

### 雾的贴图是程序生成的，不是美术资源

当前的"大团云雾"贴图和"不规则遮罩"贴图都是程序生成的。程序用多个径向渐变团块叠加来模拟云雾形态，比之前的 Perlin 噪声更像云，但仍然缺少手绘贴图的有机细节。理想方案：

- 由 UI 设计师输出 1 张 1024×1024 或更大的无缝衔接"大团云雾"贴图（白底，带 Alpha 通道/透明度）
- 由 UI 设计师输出 1 张 256×256 的"不规则云朵边缘"遮罩图（中间实心，边缘呈絮状、散落状）
- 替换 `generateFogCloudTexture()` 和 `generateSpotlightMask()` 的输出即可，代码架构已预留

### 雾的体积感仍然有限

3 层视差平移虽然速度不同，但每层都是同一张贴图的平铺。真正的雾有前后景深——近处的雾浓且移动快，远处的雾淡且移动慢。目前只有速度和不透明度的差异，缺少"远近雾团形状不同"的层次感。如果设计师能提供 2-3 张不同形态的云雾贴图，分别用于不同层，效果会更好。

### 回涌没有冷却机制

产品设计官建议引入冷却机制——鼠标离开特定区域 2.5 秒后，该区域才开始执行回涌渲染。目前回涌是全局均匀的，没有空间上的延迟。这个功能尚未实现。

## 分层结构

```
┌─────────────────────────────────┐
│  #stars-canvas (z:0)            │  星空背景
├─────────────────────────────────┤
│  #noise (z:9999, overlay)       │  噪点纹理
├─────────────────────────────────┤
│  #viewport (z:1)                │
│   └─ #world                     │  无限画布，transform 驱动
│       └─ .island × 7            │  浮岛 DOM 元素（永远原始外观）
├─────────────────────────────────┤
│  #fog-canvas (z:5)              │  迷雾层（Canvas，贴图式）
├─────────────────────────────────┤
│  #spotlight-glow (z:7)          │  鼠标暖色光晕（CSS）
├─────────────────────────────────┤
│  #focus-overlay (z:10)          │  聚焦时的径向渐变 + 退出点击区
├─────────────────────────────────┤
│  #connection-canvas (z:15)      │  关联线
├─────────────────────────────────┤
│  #focus-layer (z:20)            │  聚焦卡片
│   └─ .focus-card                │  独立于 #world，不受 blur 影响
└─────────────────────────────────┘
```

## 数据结构

```javascript
{
  id: 'emergence',
  title: '涌现',
  scale: 5,                  // 1-5，决定浮岛尺寸

  fullImage: 'url',          // 大图（聚焦时懒加载）

  promptAnalysis: '...',     // 提示词

}
```

## 替换美术贴图

当设计师提供贴图资源后：

1. 将"大团云雾"贴图保存为 `fog-cloud.png`
2. 将"不规则遮罩"贴图保存为 `spotlight-mask.png`
3. 修改 `generateFogCloudTexture()` 改为加载图片：

```javascript
function loadFogCloudTexture() {
  const img = new Image();
  img.src = 'fog-cloud.png';
  return img; // 需要等 onload 后再使用
}
```

4. 修改 `generateSpotlightMask()` 同理
5. 雾的绘制逻辑中，`drawImage(fogCloudTex, ...)` 和 `drawImage(spotlightMaskTex, ...)` 无需改动
